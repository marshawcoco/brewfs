核心设计原则

Data-First, Metadata-Later

数据先上传到 S3，确认成功后，再通过 Redis 事务提交切片元数据。崩溃场景下最多留下孤儿 block，由 sliceRefs GC 清理，绝不丢已提交数据。

Append-Only Slice Log

每个 64MB chunk 是一个 Redis List，新切片永远 RPUSH 到末尾。从不修改已有元素，压缩通过 LTRIM+LPUSH 原子替换整个列表。

5-Level Cache Hierarchy

Redis CSC → OpenFile → Memory Pages → Disk Cache Files → Object Store。每一层独立管理，上层命中率决定性能天花板。

压缩触发条件

条件类型行为
numSlices % 100 == 99非强制后台 goroutine，不阻塞写
numSlices > 350非强制同上
numSlices ≥ 2500强制阻塞写入，等待完成

压缩算法步骤

1. doRead() → LRANGE 获取所有切片 2. skipSome() → 跳过头部 ≥1MB 的大切片 3. compactChunk() → buildSlice() 二叉树合并 └─ cut(pos) 分裂 → in-order traversal → 去空洞 4. NewSlice() → 分配新 slice ID 5. 后台: 读旧切片 → 物理合并为大 object → PUT S3 6. doCompactChunk() → CAS (LTRIM+LPUSH) 原子替换
